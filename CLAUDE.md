## Commands

```bash
pnpm dev          # Start dev server at localhost:3000
pnpm build        # Production build (also runs fumadocs-mdx codegen)
pnpm lint         # Run ESLint
pnpm types:check  # Run fumadocs-mdx codegen + Next.js typegen + tsc
```

`fumadocs-mdx` must run before TypeScript can resolve types from `collections/server` — it auto-runs on `postinstall` and as part of `types:check`.

## Architecture

This is a **Next.js 16 + Fumadocs** documentation site. The goal is to display `Masterdoc.md` (a long personal document by the site owner) as a single scrollable, navigable page.

### Content pipeline

- `content/docs/` — where Fumadocs reads `.mdx` files from; this is where page content lives
- `source.config.ts` — defines the `docs` collection using `fumadocs-mdx`; `includeProcessedMarkdown: true` enables the getText('processed') helper
- `.source/` — auto-generated by `fumadocs-mdx` (do not edit manually)
- `src/lib/source.ts` — wraps the generated collection in a Fumadocs `loader`, exported as `source`

### Routing

| Route | File | Purpose |
|-------|------|---------|
| `/` | `src/app/(home)/page.tsx` | Landing page |
| `/docs/[[...slug]]` | `src/app/docs/[[...slug]]/page.tsx` | Docs page renderer |
| `/docs` layout | `src/app/docs/layout.tsx` | Wraps with `DocsLayout` + sidebar tree |
| `/api/search` | `src/app/api/search/route.ts` | Fumadocs search API |
| `/og/docs/[...slug]` | OG image generation per page |
| `/llms.txt`, `/llms-full.txt` | LLM-readable content routes |

### Key config

- `src/lib/shared.ts` — `appName`, route constants, and `gitConfig` (currently points to `fuma-nama/fumadocs` — update to actual repo)
- `src/lib/layout.shared.tsx` — `baseOptions()` used by both docs and home layouts
- `src/components/mdx.tsx` — custom MDX component overrides passed to every page

### Current intent

The site is a personal portfolio/biography rendered through Fumadocs' standard MDX pipeline. Content lives in `content/docs/` as `.mdx` files. The goal is a single long scrollable page with Fumadocs' built-in TOC providing in-page anchor navigation. `content/docs/index.mdx` is the primary page.
